**Error code to i18n key mapping function** (define inside the component or as a module-level helper):
```typescript
function mapAuthError(code: string, message: string, status: number, t: TFunction): string {
  if (status === 429) return t('auth.errorRateLimited');
  switch (code) {
    case 'auth.invalid_credentials': return t('auth.errorInvalidCredentials');
    case 'auth.email_taken':
      return message.includes('SSO') ? t('auth.errorEmailTakenSso') : t('auth.errorEmailTaken');
    case 'auth.password_too_long': return t('auth.errorPasswordTooLong');
    case 'auth.invalid_input': return t('auth.errorInvalidInput');
    case 'auth.account_disabled': return t('auth.errorAccountDisabled');
    default: return t('auth.errorUnexpected');
  }
}
```

**Form submit handler (`handleLogin`):**
1. Prevent default, set `loading = true`, clear `error`
2. Try/catch block:
   - `const res = await apiFetch('/auth/login/local', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ email, password }) })`
   - If `res.ok`: call `navigate('/')` then `window.location.reload()` (to re-trigger AuthProvider /auth/me fetch with new cookies)

   - Catch block (network error): set `error = t('auth.errorNetworkError')`
3. Finally: set `loading = false`
